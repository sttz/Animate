enum Sttz.Tweener.TweenCompletedBy

How a tween was completed.

This is a flags enumeration and some of its fields can occur in combination, e.g. if the tween was stopped, finished or canceled, the value can also contain the Overwrite flag to indicate the completion was caused by the tween being overwritten.

To properly check for a flag in the CompletedBy value you need to use binary-and:

if ((tween.CompletedBy & TweenCompletedBy.Stop) > 0) {
// Tween was stopped (overwritten or not)
}
if ((tween.CompletedBy & TweenCompletedBy.Overwrite) > 0) {
// Tween was overwritten (stopped, finished or cancelled)
}
See Also
TweenEventArgs.CompletedBy, ITween.CompletedBy
Enumerator:
Undefined 

Undefined value.

The value has not yet been set, e.g. because the tween has not been completed yet.

Stop 

Tween was stopped (left at its current value).

Finish 

Tween was finished (set to its end value).

Cancel 

Tween was canceled (set to its start value).

Complete 

Tween completed regularly by reaching its end.

Overwrite 

Tween was overwritten (always in combination with Stop, Finish or Cancel).

enum Sttz.Tweener.TweenOverwrite

Tween overwriting automatically stops existing tweens and avoids conflicting tweens to be created.

The TweenOverwrite enumeration holds different flag options that can be combined to define the exact overwrite behavior. E.g. TweenOverwrite.Default is actually (OnStart | Stop | Overlapping).

Some fields in TweenOverwrite are mutually exclusive:

  • OnInitialize and OnStart
  • Stop, Finish and Cancel
  • All and Overlapping
Enumerator:
Undefined 

Undefined overwriting. The parent scope's overwrite setting will be used in this case.

None 

Turn off overwriting. You'll have to manually call Animate.Stop, Animate.Finish or Animate.Cancel to avoid conflicting tweens.

OnInitialize 

Overwrite during initialization. This is usually during the next frame after the tween was created or when ITween.Validate is called.

OnStart 

Overwrite when starting the tween. The tween is started before it begins animating.

Stop 

Stop tweens that are being overwritten, leaving them at their current value.

Finish 

Finish tweens that are being overwritten, setting them to their target value.

Cancel 

Cancel tweens that are being overwritten, setting them to their initial value.

All 

Overwrite all tweens with the same target and property.

Overlapping 

Overwrite tweens with the same target and property that are actually overlapping the current tween. E.g. a tween with a start delay that doesn't interfere with the current tween is not overwritten.

Default 

Default overwrite preset. Overwrite on start, stopping all overlapping tweens.

Immediate 

Immediate overwrite preset. Overwrite on initialize, stopping all tweens with the same target and property.

enum Sttz.Tweener.TweenState

States of a tween.

Enumerator:
Unused 

The tween has not yet been set up using it's M:Tween{T}.Use method. You should only see tweens in this state if you create an instance or load one from the TweenPool. Usually, you want to use the methods on Animate, TweenGroup or Tween to create instances.

Uninitialized 

A not yet initialized tweens. The tween has been set up with its parameters but not yet initialized. Tweens are initialized during the next frame after their creation or by calling ITween.Validate.

Waiting 

The tween is waiting to start, e.g. because ITweenOptions.StartDelay has been set.

Tweening 

The tween is currently animating.

Complete 

The tween has completed, either normally or by being stopped, finished or cancelled.

Error 

The tween has encountered an error. See ITween.Error for the error message.

enum Sttz.Tweener.TweenTiming

Tween timing defines when a tween is udpated and what time it's being calcualted with.

Enumerator:
Undefined 

Undefined timing. The parent scope's timing is used in this case.

Update 

Update the tween during Unity's update loop (best for visuals).

Only one of Update, FixedUpdate or LateUpdate can be set at a time.

FixedUpdate 

Update the tween during Unity's fixed update loop (best when animating properties interacting with physics).

Only one of Update, FixedUpdate or LateUpdate can be set at a time.

LateUpdate 

Update the tween during Unity's late update loop (best for overwriting other animations).

Only one of Update, FixedUpdate or LateUpdate can be set at a time.

DefaultTime 

Use Unity's default time (i.e. Time.time). Tweens running with the default time are affected by Time.timeScale.

///

Only one of DefaultTime or RealTime can be set at a time.

RealTime 

Use Unity's real time (i.e. Time.realtimeSinceStartup. Tweens running with real time are unaffected by Time.timeScale.

Only one of DefaultTime or RealTime can be set at a time.

Default 

Default preset. Update tweens during the update loop and use the default time.

Physics 

Physics preset. Update tweens during the fixed update loop and use the default time.

Menu 

Menu preset. Update tweens during the update loop and use real time. This allows tweening while the game is paused by seetting Time.timeScale to 0.
